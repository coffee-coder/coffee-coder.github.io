关系挖掘

标签（空格分隔）： 关系挖掘

概述

关系挖掘以商品数据，商品的关系数据、外网关系数据为基础数据源，进行不同意图的不同关系挖掘，例如品类、品牌、款式意图的同义、上下位、搭配关系。

同义关系

数据源

1. i2i同款数据源
2. 商品 + 属性数据
3. 商品详情页数据
4. 用户行为数据

算法

1. 背景: 训练样本和测试样本特征稀疏
2. 全量商品数据源 
    1. auction_info_all
    2. query_auc_entity
3. 字段
    nid ; title ; topcate ; cate
4. 提高词向量覆盖率。
5. 提高标题属性特征覆盖率。

问题

查询点击特征比较稀疏

解决方案

1. 海选 + 精选
2. 每个词都有一个候选同义词

上下位关系挖掘

数据源

1. 商品数据
2. 用户行为数据

算法

1 商品标题属性校验

    a. 查询【泡泡袖】，查看属性项中包含【泡泡袖】的属性名。得到
    { pname, pvalue }，{ 泡泡袖，袖型 }
    b. 寻找查询【泡泡袖】商品中出现的其他袖型。
  • 采取频繁二项集算法进行挖掘。人工审核。
